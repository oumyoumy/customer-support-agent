# Agent Spec

## Overview

A graph-based customer support workflow agent for a shipping company built with
ADK 2.0 `Workflow`. The agent uses a multi-node directed graph to:

1. **Classify** every incoming user query as either `shipping`-related or `unrelated`.
2. **Route** to the appropriate handler based on the classification.
3. **Answer** shipping questions via a knowledgeable FAQ agent — or **decline** politely for unrelated queries.

## Example Use Cases

| User Query | Expected Behaviour |
|---|---|
| "How much does overnight shipping cost?" | Routed to FAQ agent → explains overnight rate ($24.99) |
| "Where is my package #TRK123?" | Routed to FAQ agent → explains tracking at track.ourshipping.com |
| "How do I return an item?" | Routed to FAQ agent → explains 30-day return policy |
| "What's the weather like today?" | Routed to decline node → polite out-of-scope message |
| "Can you recommend a laptop?" | Routed to decline node → polite out-of-scope message |

## Workflow Graph

```
START → classifier_agent → route_query ─┬─(shipping)──→ shipping_faq_agent → emit_faq_response
                                         └─(unrelated)─→ decline_unrelated
```

## Agent Nodes

| Node | Type | Purpose |
|---|---|---|
| `classifier_agent` | `LlmAgent` | Classifies query as `"shipping"` or `"unrelated"` |
| `route_query` | Function node | Reads state, emits routing `Event` |
| `shipping_faq_agent` | `LlmAgent` | Answers shipping questions with FAQ knowledge |
| `emit_faq_response` | Function node | Wraps FAQ answer as visible `Content` event |
| `decline_unrelated` | Function node | Returns polite out-of-scope message |

## Tools Required

No external API calls. All knowledge is embedded in the LlmAgent instructions.
FAQ topics covered: rates, tracking, delivery, returns, lost/damaged packages.

## Constraints & Safety Rules

- The workflow MUST NOT answer questions unrelated to shipping.
- The decline message must always offer to help with shipping topics.
- No PII should be stored in state beyond the current session.

## Success Criteria

- Shipping queries always route to `shipping_faq_agent`.
- Non-shipping queries always route to `decline_unrelated`.
- FAQ answers are factually consistent with the embedded knowledge.
- Decline responses are polite and offer a redirect.
